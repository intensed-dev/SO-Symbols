# Security Policy

## Supported Versions

Only the latest release of San Orien Symbols is officially supported.

Older versions may not receive fixes.

---

## Reporting a Security Issue

Please **do not** create a public GitHub Issue for security vulnerabilities.

Instead, report them privately.

Preferred contact:

* GitHub Security Advisories (if enabled)
* Discord: **@intense.dev**

Include:

* A description of the issue
* Steps to reproduce
* Affected files or icons
* Proof of concept (if applicable)
* Suggested fix (optional)

---

## Response Process

After receiving a report, I will:

1. Confirm receipt.
2. Investigate the issue.
3. Develop a fix if necessary.
4. Publish the fix in a future release.
5. Credit the reporter unless anonymity is requested.

---

## Scope

Examples include:

* Supply chain issues
* Malicious dependencies
* Build pipeline vulnerabilities
* Compromised release artifacts
* Security-related implementation flaws

Reports unrelated to security should be submitted using the normal Issue templates.

Thank you for helping keep San Orien Symbols secure.
